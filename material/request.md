目标
根据所学的高级程序设计（c语言编程）方法，设计并实现基于Windows控制台的简易贪吃蛇游戏，熟练掌握C语言编程语法、程序设计逻辑、集成开发环境应用、AI编程等技能。
本项目分为两大部分：
• 第一部分：根据文档要求，观看指导视频，理解代码的实现原理，完成游戏的框架搭建，实现基本的游戏逻辑，使得程序能够正确运行。

具体要求如下：
在CMD中设计一个游戏范围边界，食物在边界内随机刷新，贪吃蛇在窗口中完成吃食物动作，每吃一个食物，蛇长度加1，得分加1，当撞到自身或边界游戏结束。

指针与地址
1、指针基础
通过指针，可以简化一些 C 编程任务的执行，还有一些任务，如动态内存分配，没有指针是无法执行的。 项目中蛇的链表实现、游戏状态修改、函数传参、内存管理全部依赖指针。
指针变量即专门存储“内存地址”的变量，通过两个符号来操作：
• &符号：取地址，返回值是对应类型的指针，语法如下：&变量名
• *符号：通过指针存储的地址，找到对应的变量，操作该变量的“数据本身”，语法如下：*指针变量名
C 语言中，结构体指针访问内部成员时，(*p).成员名 可简化为 p->成员名，二者完全等价，语法更简洁。
2、指针传参
游戏通过指针传参来修改蛇/游戏过程中的状态变化，通过传参分为值传递和地址传递两种方式，区别如下：
传递方式	描述
值传递	传的是变量的副本，相当于把原变量的值赋给了新变量
地址传递	传的是变量的内存地址，可以直接修改变量的值
项目中要使用 地址传递 ，才能使得在不同文件中引用的变量在计算中发生变化。
为防止出现野指针，使用完后要及时置空NULL。 在使用指针的时候，要先判断是否为空指针，为空指针直接返回，例如：

单向链表
贪吃蛇的身体是单向链表，而链表则是一个指针自引用的结构体，指针指向下一个节点的内存地址，蛇在移动时，蛇身跟随前一节移动，就要把上一节的内存地址给本节存储，本节的内存地址给下一节……依次循环，完成整个蛇身的移动。 实现方式如下：

蛇头的位置根据方向改变。 同样是将新蛇头的内存地址给旧的蛇头，旧的蛇头地址给蛇身，原来的蛇身内存地址则指向空。 这样就完成了整个蛇的移动。

Windows 控制台编程
Windows控制台编程是实现游戏交互和界面的核心，通过原生Windows API 或标准库可以实现控制台句柄操作、键盘输入、光标控制等功能，进一步实现贪吃蛇游戏的过程。
• 控制台句柄操作：控制台句柄是Windows系统中用于访问控制台输入和输出缓冲区的关键标识符，允许程序直接与控制台交互，例如读取输入、写入输出或修改显示属性。 GetStdHandle 函数提供用于检索与进程关联的标准输入（STDIN）、标准输出（STDOUT）和标准错误（STDERR）句柄的机制。 GetStdHandle就好比获得Windows控制台的操作授权令牌，只有获得了控制权，才能在控制台中进行坐标定位、光标控制等操作。
• 光标控制：通过SetConsoleCursorPosition控制光标，可以实现在Windows控制台中的任意位置进行句柄相关的操作。 在贪吃蛇游戏中，我们需要通过移动光标位置来绘制游戏的边框、蛇、食物以及基本的信息提醒（例如分数、移动速度等）。 其次，在窗口中，需要设置光标的属性bVisible为False，表示不可见，以免影响游戏时的视觉体验。
• 无阻塞键盘输入：通过函数_kbhit()+_getch()的组合来实现蛇的实时控制，_kbhit()检测控制台是否有按键按下，有则返回非 0，无则返回 0，可以实时接收蛇的转向信号，而_getch()则是无回显读取按下的具体按键值，通过键值与方向的映射来确定蛇的转向信息。
• 清屏API：游戏画面的渲染过程实际上是通过不断地循环绘制来完成的，蛇长的更新、分数的更新，食物的刷新都需要实时绘制，因此需要在每一次画面更新之前，通过封装system("cls")来实现对前一个画面进行清屏操作。
• 系统时间API：系统时间API可以用来记录食物刷新时间，控制蛇速，以及在随机刷新食物时，作为随机种子生成伪随机数，随机生成食物坐标。 常用的有：Sleep()、GetTickCount()、clock()、time()。
贪吃蛇游戏逻辑
• 坐标运算与边界检测：在设计业务逻辑时，要保证蛇/食物的坐标不要超过游戏边界的范围，比如蛇的坐标（x，y），游戏宽width，高height，那么坐标范围应该为0≤x<width、0≤y<height，避免越界。
• 随机数的生成：通过时间函数来初始化随机种子，这样可以在调用rand()时生成不重复的数值，通常用来随机食物刷新，通过srand(time(NULL)) + rand()组合方式实现。
• 碰撞检测：当蛇碰撞自身或者碰撞边界时，游戏触发结束，这里涉及到两类逻辑： • 蛇自身碰撞：检测蛇头的坐标是否与任意蛇身节点的坐标相等，相等则出现碰撞。 • 碰撞边界：检测蛇头的坐标是否超出了边界的最左/右/上/下面，超出或正好相等则出现碰撞。
• 游戏状态管理：判断游戏是否结束（触发积分榜）、分数更新、蛇头方向等。
• 核心运动逻辑：通常使用头增尾删法来实现，不吃食物仅移动时，蛇头指针指向新位置，依次将蛇身节点指针由尾节指向前节；吃食物时，由原尾节点指向新尾节点，实现增长。
• 关卡差异化设计：在第一关基础关的基础上，需要在移动速度，游戏玩法，加减分机制上进行差异化设计，达到游戏的趣味性。
• 进阶安全编程：在项目开发过程中，可能会用到高级编程的方法，例如安全编程技巧，加入一些必要的判断，避免空指针、野指针造成内存泄漏，对非法输入处理，采用安全字符串操作等。
